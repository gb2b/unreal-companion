# P7.1 Party Mode Complet

## Objectif

Étendre le Party Mode pour supporter plus d'agents, des discussions parallèles, et une synthèse avancée avec action items.

## Prérequis

- [x] P2.8-lite - Party Mode simplifié

## Améliorations

### Par rapport à la version Lite

| Aspect | Lite | Full |
|--------|------|------|
| Nombre d'agents | 2-3 max | Jusqu'à 5 |
| Tours | Séquentiels | Parallèles possibles |
| Synthèse | Simple | Avec action items |
| Auto-tasks | Non | Oui |
| Historique | Session | Persistant |

### Nouvelles Fonctionnalités

1. **Agents parallèles** : Plusieurs agents peuvent répondre en même temps
2. **Débat structuré** : Modération automatique pour éviter les répétitions
3. **Action items** : Extraction automatique des tâches à créer
4. **Historique** : Garder les sessions Party pour référence
5. **Templates** : Sessions pré-définies (brainstorm, review, planning)

### Synthèse Avancée

```yaml
summary:
  topic: "Système de progression"
  duration: "15 minutes"
  participants: [maya, alex, sam]

  key_points:
    - "Progression par connaissance, pas XP"
    - "DataAsset pour tracker les mots appris"
    - "Hubs progressifs pour le pacing"

  decisions:
    - decision: "Utiliser un DataAsset pour la progression"
      proposed_by: alex
      agreed_by: [maya, sam]

  action_items:
    - title: "Créer DA_PlayerKnowledge"
      sector: dev
      assigned: game-architect
      from_decision: 0

    - title: "Définir les hubs progressifs"
      sector: levels
      assigned: level-designer

  open_questions:
    - "Comment visualiser la progression dans l'UI ?"
    - "Combien de mots avant le premier hub ?"
```
